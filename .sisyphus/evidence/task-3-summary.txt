TASK 3: BINARY AVAILABILITY VALIDATION UTILITIES
================================================

OBJECTIVE:
Create utility functions to check for ffmpeg/sox binary availability, design 
startup validation that fails gracefully if binaries missing, implement version 
checking for compatibility requirements, and create clear error messages for 
missing dependencies.

IMPLEMENTATION SUMMARY:
======================

1. UTILITY FUNCTIONS CREATED:
   - check_ffmpeg_available() -> bool
     Returns True if ffmpeg binary is found in system PATH
   
   - check_sox_available() -> bool
     Returns True if sox binary is found in system PATH
   
   - get_ffmpeg_version() -> Optional[str]
     Retrieves ffmpeg version string via subprocess
   
   - get_sox_version() -> Optional[str]
     Retrieves sox version string via subprocess

2. AUDIOPROCESSOR METHOD UPDATED:
   - Renamed: _validate_ffmpeg_availability() → _validate_cli_tools_availability()
   - Enhanced to check both ffmpeg (required) and sox (optional)
   - Logs version information when binaries are available
   - Provides clear, actionable error messages with installation instructions

3. ERROR MESSAGES:
   When FFmpeg is missing:
   ├─ "FFmpeg binary not found in system PATH"
   ├─ "Please install FFmpeg system package"
   ├─ "On Ubuntu/Debian: sudo apt-get install ffmpeg"
   ├─ "On Fedora/RHEL: sudo dnf install ffmpeg"
   └─ "On macOS: brew install ffmpeg"
   
   When SoX is missing (warning, not fatal):
   ├─ "SoX binary not found in system PATH"
   ├─ "Some audio processing features may be unavailable"
   └─ Installation instructions for Ubuntu/Debian and macOS
